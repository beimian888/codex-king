# 系统管理模块设计

## 背景

根据参考截图，新增一个后台风格的“系统管理”模块。模块先做前端页面、导航入口和静态示例数据，重点恢复截图中的管理台结构：顶部导航进入系统管理，页面内有二级菜单，当前落点为“卡密管理”。

本次不接真实后端接口，不实现真实卡密生成、删除、用户权限校验或 CDN 数据查询。后续接接口时再扩展数据来源和权限逻辑。

## 目标

- 在默认后台布局中增加“系统管理”顶部导航入口。
- 新增 `/admin/system-management` 路由。
- 新增系统管理页面，视觉参考截图并复用现有项目深色/玻璃后台风格。
- 页面内提供左侧二级菜单：运维总览、用户管理、卡密管理、CDN 数据。
- 当前默认展示“卡密管理”内容，包括统计、排序按钮、表格和复制操作。
- 页面在桌面端接近截图布局，在移动端可正常纵向浏览，表格允许横向滚动。

## 页面结构

系统管理页由三个区域组成：

1. 标题区
   - 小标签：`ADMIN`
   - 标题：`系统管理`
   - 简短说明：用于管理卡密、用户和运行数据。

2. 主内容区
   - 左侧二级导航，当前选中“卡密管理”。
   - 右侧内容面板，先展示“卡密管理”。

3. 卡密管理面板
   - 标题：`卡密管理`
   - 说明：`生成注册卡密，用户持卡密注册后自动激活`
   - 右上角：`刷新` 按钮和权限提示。
   - 三个统计块：月卡、季卡、年卡，展示未用/已用数量。
   - 排序按钮：生成时间、档位、状态、使用时间。
   - 表格列：卡密、档位、到期时间、状态、使用者、使用时间、生成时间、备注、操作。

## 组件与状态

新增 `src/views/SystemManagement.vue`，保持单页面组件，避免为静态首版过早拆分。

组件内部状态：

- `activeSection`：当前二级菜单项，默认 `cards`。
- `sortKey`：当前排序字段，默认 `createdAt`。
- `licenseCards`：本地示例卡密数据。
- `summaryCards`：由示例数据计算月卡、季卡、年卡未用/已用数量。

交互：

- 点击左侧菜单切换当前项。非卡密项先显示简洁占位状态，避免做假功能。
- 点击排序按钮更新 `sortKey` 并重排表格。
- 点击复制按钮复制对应卡密。复制成功使用 Naive UI message 提示；浏览器不支持剪贴板时使用降级方案。
- 刷新按钮仅重置排序并提示“已刷新示例数据”，不调用接口。

## 导航与路由

- 在 `src/router/index.js` 的 `/admin` 子路由中增加 `system-management`。
- 路由名为 `SystemManagement`。
- 页面标题为 `系统管理`。
- 在 `src/layout/DefaultLayout.vue` 桌面导航和移动抽屉中增加“系统管理”入口。
- 图标优先使用已有 `@vicons/ionicons5`，选择接近设置/管理含义的图标。

## 视觉规则

- 复用现有 `var(--app-background)`、`var(--card-bg)`、`var(--app-line)`、`var(--primary-color)` 等主题变量。
- 保持产品后台密度，避免营销式大 hero。
- 卡片圆角控制在现有系统范围内，不使用夸张圆角。
- 表格状态使用清晰 badge：`已使用`、`未使用`。
- 桌面端使用两列布局，左侧菜单固定宽度；移动端改为一列，二级菜单横向/纵向都不遮挡内容。

## 错误与边界

- 剪贴板复制失败时显示错误提示，不让按钮静默失败。
- 长卡密、用户名等内容使用省略或横向滚动，避免撑破布局。
- 低宽度屏幕下表格容器横向滚动，按钮和标题自动换行。
- 非卡密管理子菜单仅展示占位说明，不承诺已有真实功能。
