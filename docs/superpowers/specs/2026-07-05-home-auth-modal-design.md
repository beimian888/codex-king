# 首页登录/注册弹窗设计

## 背景

首页已经恢复为 5176 风格的登录弹窗。用户希望在这个弹窗上增加“注册”入口，点击后弹窗本身切换为注册弹窗，而不是跳转页面或打开另一套视觉风格。

## 推荐方案

复用现有 `.login-modal`、`.login-container`、`.login-form` 弹窗结构，新增一个弹窗模式状态，例如 `authMode`。弹窗打开时默认进入登录模式；登录模式下展示用户名、密码、登录按钮，并在登录按钮附近展示“注册”按钮。点击“注册”按钮后，将 `authMode` 切换为注册模式。

注册模式沿用同一套 5176 风格容器、遮罩、关闭按钮和输入框样式。标题显示“注册”，输入框依次为用户名、密码、确认密码、卡密，底部提交按钮显示“注册”。

## 交互

打开首页登录弹窗时，默认显示登录表单。点击登录表单中的“注册”按钮后，弹窗内容切换为注册表单。遮罩点击、右上角关闭按钮、Esc 关闭逻辑保持不变。关闭后再次打开弹窗时回到登录模式。

## 组件与数据流

该改动仅影响 `src/views/Home.vue`。新增 `authMode` 响应式状态，不新增路由、不接入接口、不改变现有导航逻辑。表单提交暂时只阻止默认提交，保持当前登录弹窗的无后端行为。

## 样式

注册弹窗复用登录弹窗的视觉语言，包括暗色半透明表单、wheat 色输入框、圆角提交按钮和模糊遮罩。由于注册表单有四个输入框，需要适当调整标题和按钮间距，保证桌面和移动端都不溢出。

## 测试

更新 `test/home-login-modal.test.js`，覆盖以下行为：

- 首页登录按钮仍打开 5176 风格 `.login-modal`。
- 登录模式包含用户名、密码、登录按钮和“注册”切换按钮。
- 点击注册后应存在注册模式所需的用户名、密码、确认密码、卡密四个输入框和注册提交按钮。
- 首页不恢复 Token 导入弹窗相关引用。

同时运行现有账号导航测试和生产构建，确认本次改动没有破坏既有页面。
