# 首页登录/注册接入系统管理设计

## 背景

首页已经有 5176 风格登录/注册弹窗，系统管理已经有卡密管理和用户管理入口。当前两者互不相通：注册表单不会消耗卡密，登录表单不会校验本地用户，系统管理页面也无法看到首页注册产生的用户。

## 目标

做一个前端本地可用的接入版本：用户在首页用卡密注册后，系统管理的卡密表要显示该卡密已使用，并记录使用者和使用时间；首页登录要从同一份本地用户数据中校验账号密码；登录或注册成功后进入 `/admin/system-management`。

## 范围

- 不接真实后端接口。
- 不做真实权限系统和密码加密。
- 不改变 Token 管理、游戏账号导入和网页游戏登录流程。
- 保留首页弹窗的 5176 视觉风格。
- 系统管理继续作为后台页面，新增可读的用户管理表。

## 数据设计

新增 `src/utils/systemManagementData.js` 作为本地系统管理数据层，统一负责：

- 默认卡密数据。
- `localStorage` 读写。
- 注册用户：校验用户名、密码、确认密码、卡密存在且未使用。
- 登录用户：校验用户名和密码。
- 卡密状态更新：注册成功后把卡密标记为 `used`，写入 `user`、`usedAt`。
- 用户列表：系统管理的用户管理页读取同一份数据。

本地存储键使用 `xyzw_system_license_cards`、`xyzw_system_users`、`xyzw_system_session`，避免和现有 `registeredUsers`、`token`、`user` 等键混淆。

## 页面行为

首页弹窗：

- 登录模式填写用户名、密码。
- 注册模式填写用户名、密码、确认密码、卡密。
- 注册成功后自动创建本地系统用户、消耗卡密，并进入系统管理。
- 登录成功后进入系统管理。
- 失败时在弹窗内显示中文错误文案。

系统管理：

- 卡密管理从本地数据层读取卡密，而不是组件内硬编码。
- 用户管理从空状态升级为用户表，展示用户名、激活卡密、档位、到期时间、注册时间、最后登录时间、状态。
- 刷新按钮重新读取本地数据层，提示“已刷新系统管理数据”。

## 测试

新增 `test/home-system-management-auth.test.js`：

- 实际执行本地数据层注册/登录逻辑。
- 静态验证首页弹窗调用本地数据层并跳转系统管理。
- 静态验证系统管理读取同一份卡密和用户数据。
