# Backend SQLite System Data Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Flask + SQLite backend that becomes the source of truth for license cards, users, admins, login, registration, and admin card quotas.

**Architecture:** Add a focused Python database module for SQLite schema and business rules, then expose those operations through a small Flask API under `/api`. Keep the frontend boundary file `src/utils/systemManagementData.js`, but turn it into an async API client so `Home.vue`, `SystemManagement.vue`, router guards, and navigation keep using one shared interface.

**Tech Stack:** Python Flask, Python stdlib `sqlite3`, `unittest`, `werkzeug.security`, Vue 3 `<script setup>`, axios/fetch-compatible frontend API helpers, Node static contract tests, Vite build.

## Global Constraints

- Database file: `server/data/system.db`.
- Backend entry: `server/app.py`.
- Database module: `server/system_database.py`.
- Built-in super admin: username `111`, password `111`, role `super_admin`.
- Roles: `super_admin`, `admin`, `user`.
- License levels: `月卡`, `季卡`, `年卡`.
- License durations after activation: `月卡` 30 days, `季卡` 90 days, `年卡` 365 days.
- License cards must be generated by the backend; frontend must not supply custom card keys.
- Normal admins can only see cards they generated.
- Normal admins can create cards within per-level quota, but cannot update or delete cards.
- Super admins can create, read, update, and delete all cards and admins.
- Normal registered users cannot enter system management and cannot generate cards.
- Do not auto-import old browser `localStorage` system data in this implementation.
- Do not change game account token storage.

const SYSTEM_API_BASE = "/api";

const requestSystemApi = async (path, options = {}) => {
  const response = await fetch(`${SYSTEM_API_BASE}${path}`, {
    credentials: "include",
    headers: {
      "Content-Type": "application/json",
      ...(options.headers || {}),
    },
    ...options,
  });
  const payload = await response.json().catch(() => ({
    success: false,
    message: "服务响应格式错误",
  }));
  if (!response.ok && payload.success !== false) {
    return { success: false, message: payload.message || "请求失败" };
  }
  return payload;
};
