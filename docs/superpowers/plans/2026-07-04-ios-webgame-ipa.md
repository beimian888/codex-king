# iOS Webgame IPA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Capacitor-based iOS wrapper template that opens the bundled web game directly and can later be signed/exported as an IPA on macOS/Xcode.

**Architecture:** Add a focused Capacitor configuration whose `webDir` is `ios-webgame`, not the normal Vite `dist`. A Node preparation script validates and copies `public/web-game/asar` into `ios-webgame`, creates a tiny direct-launch `index.html`, and leaves Xcode signing/export to documented macOS steps.

**Tech Stack:** Vue/Vite project, pnpm, Node ESM scripts, Capacitor 8.4.1 (`@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`), plain Node assertion tests.

## Global Constraints

- App scope: only封装网页游戏本体，不进入 Web Helper 后台。
- Default Bundle ID: `com.example.xyzw.webgame`, and it must be replaced before real signing.
- Windows environment: can prepare Web resources and config, but cannot run Xcode archive/signing.
- macOS requirement: Xcode, Apple Developer account, certificate, provisioning profile are required for installable IPA.
- No sensitive token is embedded in the App bundle.
- Capacitor dependencies must use aligned version `8.4.1`.

Create `scripts/prepare-ios-webgame.mjs`:

```js
import {
  cp,
  mkdir,
  readdir,
  rm,
  stat,
  writeFile,
} from "node:fs/promises";
import path from "node:path";
import { fileURLToPath } from "node:url";

const scriptDir = path.dirname(fileURLToPath(import.meta.url));
const defaultRootDir = path.resolve(scriptDir, "..");

const requiredFiles = ["index.html"];
const requiredPatterns = [/^main\..+\.js$/, /^cocos2d-js-min\..+\.js$/];

async function pathExists(targetPath) {
  try {
    await stat(targetPath);
    return true;
  } catch (error) {
    if (error?.code === "ENOENT") return false;
    throw error;
  }
}

async function countFiles(targetDir) {
  const entries = await readdir(targetDir, { withFileTypes: true });
  let total = 0;

  for (const entry of entries) {
    const entryPath = path.join(targetDir, entry.name);
    if (entry.isDirectory()) {
      total += await countFiles(entryPath);
    } else if (entry.isFile()) {
      total += 1;
    }
  }

  return total;
}

export async function validateGameAssets(sourceDir) {
  const entries = await readdir(sourceDir, { withFileTypes: true });
  const fileNames = entries
    .filter((entry) => entry.isFile())
    .map((entry) => entry.name);

  for (const fileName of requiredFiles) {
    if (!fileNames.includes(fileName)) {
      throw new Error(`Missing required web game asset: ${path.join(sourceDir, fileName)}`);
    }
  }

  for (const pattern of requiredPatterns) {
    if (!fileNames.some((fileName) => pattern.test(fileName))) {
      throw new Error(`Missing required web game asset matching ${pattern} in ${sourceDir}`);
    }
  }
}

function createLaunchHtml() {
  return `<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta
      name="viewport"
      content="width=device-width, initial-scale=1, viewport-fit=cover, user-scalable=no"
    />
    <meta name="theme-color" content="#05070a" />
    <title>咸鱼之王</title>
    <style>
      html,
      body {
        width: 100%;
        height: 100%;
        margin: 0;
        overflow: hidden;
        background: #05070a;
      }

      body {
        display: grid;
        place-items: center;
        color: #e6eef8;
        font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
      }
    </style>
  </head>
  <body>
    <script>
      const target = new URL("web-game/asar/index.html", window.location.href);
      target.search = window.location.search;
      window.location.replace(target.href);
    </script>
  </body>
</html>
`;
}

export async function prepareIosWebgame(options = {}) {
  const rootDir = options.rootDir || defaultRootDir;
  const sourceRelativeDir = options.sourceRelativeDir || path.join("public", "web-game", "asar");
  const outputRelativeDir = options.outputRelativeDir || "ios-webgame";
  const sourceDir = path.resolve(rootDir, sourceRelativeDir);
  const outputDir = path.resolve(rootDir, outputRelativeDir);
  const outputGameDir = path.join(outputDir, "web-game", "asar");

  if (!(await pathExists(sourceDir))) {
    throw new Error(`Web game asset directory does not exist: ${sourceDir}`);
  }

  await validateGameAssets(sourceDir);
  await rm(outputDir, { recursive: true, force: true });
  await mkdir(outputGameDir, { recursive: true });
  await cp(sourceDir, outputGameDir, { recursive: true });
  await writeFile(path.join(outputDir, "index.html"), createLaunchHtml(), "utf8");

  const copiedFiles = await countFiles(outputGameDir);

  return {
    sourceDir,
    outputDir,
    copiedFiles,
  };
}

if (import.meta.url === `file://${process.argv[1].replace(/\\/g, "/")}`) {
  try {
    const result = await prepareIosWebgame();
    console.log(
      `[prepare-ios-webgame] Copied ${result.copiedFiles} files from ${result.sourceDir} to ${result.outputDir}`,
    );
  } catch (error) {
    console.error(`[prepare-ios-webgame] ${error.message}`);
    process.exitCode = 1;
  }
}
```

# iOS IPA 打包说明

本文说明如何把当前项目中的网页游戏封装为 iOS App，并在 macOS/Xcode 环境中导出 IPA。

## 当前交付状态

本项目会生成 Capacitor iOS 工程模板和 App 内置网页游戏资源。当前默认 Bundle Identifier 是 `com.example.xyzw.webgame`，正式签名前必须替换为 Apple Developer 账号下可用的 Bundle Identifier。

没有 Apple 证书、Signing Certificate 和 Provisioning Profile 时，不能导出可安装到真机的 IPA。Windows 环境也不能执行 Xcode 归档或签名。

不要把真实 Token、账号密钥或个人登录信息写入 App 包。初版 App 只内置网页游戏静态资源。

## Xcode 签名配置

在 Xcode 中打开 App target，进入 `Signing & Capabilities`：

1. 选择你的 Team。
2. 把 Bundle Identifier 从 `com.example.xyzw.webgame` 改为你的真实 Bundle Identifier。
3. 选择 Signing Certificate。
4. 选择或自动生成 Provisioning Profile。
5. 确认真机调试时设备已经加入该 Provisioning Profile。

如果 Xcode 报 Bundle Identifier 不可用，需要到 Apple Developer 后台创建新的 App ID，或改成账号下已有的 Bundle Identifier。

## 导出 IPA

在 Xcode 中：

1. 选择真机或 `Any iOS Device`。
2. 执行 `Product > Archive`。
3. Archive 完成后打开 Organizer。
4. 点击 `Distribute App`。
5. 按你的用途选择 Development、Ad Hoc、Enterprise 或 App Store Connect。
6. 按向导选择证书和 Provisioning Profile。
7. 点击 `Export` 导出 IPA。

Development IPA 适合自己的设备测试。Ad Hoc IPA 适合给已登记 UDID 的设备安装。App Store Connect 用于 TestFlight 或上架流程。
